# Запуск приложения в контейнере (Podman)

Инструкция для macOS с **Podman Desktop** (без Docker Desktop). Используется тот же `Dockerfile` в корне репозитория: в образе собирается фронтенд и поднимается Express на порту **8080**.

## Что нужно заранее

1. Установлен и запущен **Podman Desktop** (иконка в меню, машина в состоянии *Running*).
2. В терминале доступна команда `podman` (см. раздел про PATH ниже).

Проверка:

```bash
podman --version
podman machine list
```

Если машина не запущена:

```bash
podman machine start
```

## Если в терминале: `command not found: podman`

Podman Desktop часто ставит CLI в `/opt/podman/bin`, не добавляя его в `PATH`.

**Разовый запуск** — указывайте полный путь:

```bash
/opt/podman/bin/podman --version
```

**Постоянно** — в `~/.zshrc` добавьте строку:

```bash
export PATH="/opt/podman/bin:$PATH"
```

Выполните `source ~/.zshrc` и снова проверьте `podman --version`.

Если бинарника нет по этому пути, откройте Podman Desktop → **Settings** и найдите раздел про **CLI** / установку `podman` в PATH.

## Сборка образа

Переходите в **корень проекта** (где лежат `Dockerfile` и папки `frontend/`, `backend/`):

```bash
cd /путь/к/ikbo-12-23-morev
podman build -t ikbo-parking:latest .
```

Имя тега (`ikbo-parking:latest`) можно заменить на своё.

## Запуск контейнера

Проброс порта **8080** на хост (как в учебном задании):

```bash
podman run --rm -p 8080:8080 ikbo-parking:latest
```

- **`--rm`** — контейнер удаляется после остановки (`Ctrl+C` в терминале).
- **`-p 8080:8080`** — снаружи `localhost:8080` → внутри контейнера порт 8080.

Откройте в браузере: **http://localhost:8080** — должна открыться веб-часть, API доступно по путям `/api/...`.

## Другой порт на компьютере

Например, снаружи порт **9090**, внутри контейнера по-прежнему 8080:

```bash
podman run --rm -p 9090:8080 ikbo-parking:latest
```

Сайт: http://localhost:9090

## Переменные окружения

В образе по умолчанию заданы `PORT=8080` и `HOST=0.0.0.0`. При необходимости можно переопределить (редко нужно для этой лабораторной работы):

```bash
podman run --rm -p 3000:3000 -e PORT=3000 -e HOST=0.0.0.0 ikbo-parking:latest
```

Порт в `-p` должен совпадать с тем, что слушает процесс внутри контейнера (`PORT`).

## Сохранение данных бэкенда

Сейчас данные лежат в образе/контейнере в каталоге приложения. Чтобы привязать хостовую папку к данным в контейнере (если позже добавите том для `data/`), используйте `-v` в `podman run` — см. документацию Podman по volume.

## Типичные проблемы

| Симптом | Что сделать |
|--------|-------------|
| `command not found: podman` | PATH или `/opt/podman/bin/podman` (см. выше). |
| Ошибка подключения к сокету / machine | Запустите Podman Desktop или `podman machine start`. |
| `port is already allocated` | Занят порт 8080 другим процессом; смените левую часть `-p`, например `-p 8081:8080`. |
| Пустая страница или 404 на `/api` | Убедитесь, что контейнер запущен без ошибок в логах; образ собран из актуального `Dockerfile`. |

## Краткая шпаргалка

```bash
cd /путь/к/ikbo-12-23-morev
podman build -t ikbo-parking:latest .
podman run --rm -p 8080:8080 ikbo-parking:latest
# браузер: http://localhost:8080
```

Команды совпадают с Docker по смыслу: везде, где в методичке написано `docker`, можно подставить `podman` (при установленном и доступном CLI).
